Proposed Architecture

Two-layer design: Python core + HTML viewer, adapts to Claude Code or Claude Desktop

System Overview

Architecture Diagram
User Input → "Compare these 4 videos about GPT-5"
Core Layer (Python Scripts)
1. compare_videos.py
Accept multiple URLs/IDs
2. get_transcript.py
Fetch all transcripts (existing)
3. capture_frames.py
ffmpeg frame extraction at timestamps
4. Output: comparison_data.json
All metadata, transcripts, screenshots (base64), analysis structure
Viewer Layer (HTML + JS)
viewer.html
Self-contained single file
Dashboard tab
Summary, topic map, stats
By Topic / By Video toggle
Two views of the same data
Screenshot lightbox + on-demand capture
Click timeline → API call (Code) or pre-baked (Desktop)
Claude Code Mode
compare_server.py (Flask)
Serves viewer + /api/screenshot endpoint
On-demand frame capture via ffmpeg
Claude Desktop Mode
Pre-captures all key frames
Embeds as base64 in JSON
Outputs viewer as HTML artifact

New Files to Create

scripts/
compare_videos.py — orchestrator
capture_frames.py — ffmpeg wrapper
compare_server.py — local Flask server
viewer/
viewer.html — self-contained dashboard
(inline CSS + JS, reads comparison_data.json)

Dependencies Added

• ffmpeg — frame extraction (must be installed separately)
• flask — local server for Claude Code mode (pip install)
• No Node.js or npm required
