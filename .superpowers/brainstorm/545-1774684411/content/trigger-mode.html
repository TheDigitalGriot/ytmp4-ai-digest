How should the video comparison feature be triggered?

This determines the overall architecture — standalone app vs Claude-integrated vs hybrid

A

Standalone Web App

A self-contained local web app with its own UI. You open it in the browser, paste YouTube URLs, and it orchestrates everything — calling the Python scripts behind the scenes via a local API server. Works independently of Claude.

B

Claude-Driven Workflow

You tell Claude "compare these videos: URL1, URL2, URL3" and Claude runs the scripts, generates the analysis, then outputs results as Markdown in the terminal. No separate web UI — Claude handles everything conversationally.

C

Hybrid: Claude + Interactive Viewer

Claude triggers the workflow conversationally ("compare these videos..."), runs the scripts to fetch transcripts and screenshots, generates the cross-video analysis, then serves results as an interactive HTML page you can explore — with clickable timestamps, side-by-side screenshots, and comparison views.
